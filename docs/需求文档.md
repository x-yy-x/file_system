# 文件系统管理项目 — 需求规格说明书

## 1. 项目概述

### 1.1 项目背景

操作系统是计算机系统的核心系统软件，文件系统是操作系统的重要组成部分，负责管理外存上的数据，提供文件的存储、检索、共享和保护等功能。本课程项目旨在通过模拟实现一个简单的文件系统，加深对操作系统文件管理原理的理解。

### 1.2 项目目标

| 目标                       | 描述                                       |
| -------------------------- | ------------------------------------------ |
| 理解文件存储空间管理       | 掌握文件在磁盘上的组织方式与分配策略       |
| 掌握文件物理结构与目录结构 | 深入理解文件的逻辑结构与物理结构的映射关系 |
| 实现简易文件系统           | 从零构建具备基本操作能力的文件系统         |
| 加深实现过程理解           | 通过编码实践巩固操作系统理论知识           |

---

## 2. 总体需求

### 2.1 基本要求

- 在内存中开辟一块空间作为**虚拟文件存储器**（模拟磁盘），在此之上实现一个简单的文件系统；
- 文件系统退出时，需将当前文件系统的内容**持久化保存到磁盘文件**中；
- 文件系统启动时，能够从磁盘文件中**恢复**上一次的文件系统状态。

### 2.2 开发环境

| 项目         | 要求                                |
| ------------ | ----------------------------------- |
| 操作系统平台 | 自选（Windows / Linux / macOS）     |
| 开发语言     | 自选（推荐 Java、C/C++、Python 等） |
| 开发工具     | 自选 IDE 或编辑器                   |

## 3. 功能需求

### 3.1 文件存储空间管理

**需求描述：**

实现对虚拟磁盘空间的分配与回收。磁盘空间应按逻辑划分为若干**盘块**，系统需跟踪每个盘块的分配状态。

**可选的实现方案：**

| 方案                | 说明                                                                                |
| ------------------- | ----------------------------------------------------------------------------------- |
| 显式链接结构（FAT） | 文件分配表（File Allocation Table），每个文件对应一个盘块链，FAT 表中记录下一盘块号 |
| 索引分配            | 每个文件对应一个索引块，其中列出该文件的所有盘块号                                  |
| 混合索引            | 类 UNIX 系统的多级索引结构                                                          |

### 3.2 空闲空间管理

**需求描述：**

管理虚拟磁盘中尚未分配的盘块，分配时从中取出空闲块，回收时重新加入。

**可选的实现方案：**

| 方案             | 说明                                                       |
| ---------------- | ---------------------------------------------------------- |
| 位图法（Bitmap） | 用一组二进制位表示每个盘块的占用状态，0 为空闲，1 为已分配 |
| 空闲链表法       | 将所有空闲盘块链接成一个链，分配时从头取，回收时挂回       |
| 成组链接法       | UNIX 系统中使用的空闲块管理方式                            |

### 3.3 目录结构

**需求描述：**

- 采用**多级目录结构**（树形目录），支持用户在不同目录下组织文件；
- 每个目录项应至少包含以下信息：

| 字段     | 说明                                 |
| -------- | ------------------------------------ |
| 文件名   | 文件或子目录的名称                   |
| 物理地址 | 文件在虚拟磁盘中的存储位置（盘块号） |
| 长度     | 文件大小（字节或盘块数）             |

- 支持相对路径和绝对路径的解析。

### 3.4 文件系统操作

文件系统应提供以下操作接口（命令行或图形界面）：

#### 3.4.1 格式化

- `format` — 初始化虚拟磁盘，创建空的根目录，重置空闲空间管理结构。

#### 3.4.2 目录操作

| 操作             | 功能                                   |
| ---------------- | -------------------------------------- |
| `mkdir <路径>` | 创建子目录                             |
| `rmdir <路径>` | 删除子目录（需为空目录）               |
| `ls [路径]`    | 显示指定目录的内容（文件和子目录列表） |
| `cd <路径>`    | 更改当前工作目录                       |
| `pwd`          | 显示当前工作目录路径                   |

#### 3.4.3 文件操作

| 操作                          | 功能                            |
| ----------------------------- | ------------------------------- |
| `create <文件名>`           | 创建文件                        |
| `open <文件名>`             | 打开文件（返回文件描述符/句柄） |
| `close <文件描述符>`        | 关闭文件                        |
| `write <文件描述符> <数据>` | 向文件写入数据                  |
| `read <文件描述符> <大小>`  | 从文件中读取指定大小的数据      |
| `delete <文件名>`           | 删除文件（释放所占盘块）        |

#### 3.4.4 附加操作（可选扩展）

| 操作                         | 功能                   |
| ---------------------------- | ---------------------- |
| `rename <旧名> <新名>`     | 重命名文件或目录       |
| `copy <源路径> <目标路径>` | 复制文件               |
| `move <源路径> <目标路径>` | 移动文件               |
| `stat <文件路径>`          | 显示文件详细信息       |
| `tree [路径]`              | 以树状结构显示目录层次 |

## 4. 数据持久化需求

### 4.1 保存机制

- 文件系统退出时，将**整个虚拟磁盘**（包括超级块、目录结构、FAT 表/位图、文件数据等）序列化写入磁盘文件；
- 保存格式建议采用二进制文件，保证读写效率与完整性。

### 4.2 恢复机制

- 文件系统启动时，检查是否存在之前保存的磁盘映像文件；
- 若存在，则将其全部加载回内存，恢复文件系统的完整状态；
- 若不存在，则创建一个空的虚拟磁盘并初始化。

## 5. 非功能需求

### 5.1 用户界面

- 提供清晰的**命令行交互界面（CLI）** 或**图形用户界面（GUI）**；
- 所有操作应有明确的提示信息和反馈（成功/失败/错误原因）；
- 显示当前工作目录路径作为命令提示符（如 `root/home/>`）。

### 5.2 错误处理

- 对非法操作（如删除非空目录、打开不存在的文件、写入超过磁盘容量等）应给出明确的错误提示；
- 不能因用户操作错误导致程序崩溃。

### 5.3 健壮性

- 支持合理的文件命名规则（如支持字母、数字、下划线，限制特殊字符）；
- 文件名长度应有合理限制（如 ≤ 32 字符）；
- 路径深度应有合理上限。

### 5.4 性能（可选）

- 若磁盘较大，应考虑查找效率（如使用缓存、哈希目录等优化）。

## 8. 附录：建议的架构设计

建议按以下模块组织代码：

```
+---------------------------+
|      CLI / UI 层          |   ← 命令解析、交互界面
+---------------------------+
|      文件操作接口层        |   ← open / close / read / write / delete
+---------------------------+
|      目录管理层            |   ← 多级目录、路径解析、目录项管理
+---------------------------+
|      文件分配层            |   ← FAT / 索引分配、盘块管理
+---------------------------+
|      磁盘管理层            |   ← 虚拟磁盘的读写、持久化加载/保存
+---------------------------+
|      空闲空间管理层         |   ← 位图 / 空闲链表
+---------------------------+
|      物理存储层            |   ← 内存缓冲区（模拟磁盘块）
+---------------------------+
```
