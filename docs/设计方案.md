## 1. 整体架构设计

### 1.1 系统分层架构

采用 **前后端分离** 架构，后端 Java 承载文件系统核心逻辑，前端 Vue 提供图形化交互界面，CLI 内置于后端可直接启动。

```
┌──────────────────────────────────────────────────────────┐
│                   表示层 (Presentation)                   │

│  │   Vue 前端 GUI   │    │  Java 内置 CLI (控制台)      │ │
```

## 2. 核心数据结构设计

### 2.1 虚拟磁盘参数

| 参数             | 值               | 说明                                |
| ---------------- | ---------------- | ----------------------------------- |
| 虚拟磁盘总大小   | **16 MB**  | 可通过配置文件调整                  |
| 盘块大小         | **512 B**  | 基本 I/O 单位                       |
| 总盘块数         | **32,768** | 16 MB ÷ 512 B                      |
| 位图占用块数     | **8 块**   | 32,768 ÷ 8 ÷ 512，向上取整        |
| 索引节点数量     | **4,096**  | 最大文件/目录数量                   |
| 索引节点占用块数 | **128 块** | 4,096 × 16 B ÷ 512 B              |
| 数据块起始位置   | 第 137 块        | 0: 超级块, 1-8: 位图, 9-136: i-node |

### 2.2 超级块 (SuperBlock)

大小：**512 B**（1 个盘块），位于虚拟磁盘的第 0 块。

```java
public class SuperBlock {
    int magicNumber;        // 魔数 (0xF1E5F1E5)，校验有效性
    int totalBlocks;        // 总盘块数 (32768)
    int blockSize;          // 盘块大小 (512)
    int bitmapStartBlock;   // 位图起始块号 (1)
    int bitmapBlockCount;   // 位图占用块数 (8)
    int inodeStartBlock;    // i-node 起始块号 (9)
    int inodeBlockCount;    // i-node 占用块数 (128)
    int dataStartBlock;     // 数据区起始块号 (137)
    int totalInodes;        // i-node 总数 (4096)
    int freeBlockCount;     // 空闲块数
    int freeInodeCount;     // 空闲 i-node 数
    int rootInode;          // 根目录 i-node 号 (0)
}
```

### 2.3 位图 (Bitmap)

大小：**8 个盘块**（4,096 字节 = 32,768 位，正好对应每个盘块）。

```
位图布局（每字节对应 8 个盘块）：
Byte 0: Block 0-7   的分配状态
Byte 1: Block 8-15  的分配状态
...
Byte 4095: Block 32760-32767 的分配状态

位值：0 = 空闲，1 = 已分配
```

### 2.4 索引节点 (i-node)

每个 i-node 固定大小：**64 B**，一块可存放 **8 个 i-node**。
系统共 4,096 个 i-node，占用 128 个盘块。

```
i-node 结构（16 字节对齐优化）：
┌─────────────────────────────────────────────────┐
│  字段            大小    说明                    │
├─────────────────────────────────────────────────┤
│  inodeNumber     4 B     i-node 编号 (0-4095)   │
│  fileType        1 B     0=空闲, 1=文件, 2=目录  │
│  linkCount       1 B     链接计数                │
│  fileSize        4 B     文件大小 (字节)          │
│  createTime      4 B     创建时间戳              │
│  modifyTime      4 B     修改时间戳              │
│  direct[10]      40 B    10 个直接盘块指针       │
│  singleIndirect  4 B     一级间接块指针          │
│  doubleIndirect  4 B     二级间接块指针          │
│  填充            2 B     对齐到 64 B            │
└─────────────────────────────────────────────────┘
```

**文件最大大小计算：**

| 级别                                | 容量               |
| ----------------------------------- | ------------------ |
| 直接块 (10 × 512 B)                | 5,120 B (5 KB)     |
| 一级间接 (1 × 128 × 512 B)        | 65,536 B (64 KB)   |
| 二级间接 (1 × 128 × 128 × 512 B) | 8,388,608 B (8 MB) |
| **合计最大**                  | **~8.06 MB** |

> 一个盘块号占 4 字节，每块可存 512/4 = 128 个盘块号。

### 2.5 目录项 (DirectoryEntry)

每个目录项固定大小：**48 B**，一块可存放 **10 个** 目录项。

```java
public class DirectoryEntry {
    int inodeNumber;       // 4 B，i-node 编号
    String fileName;       // 40 B，文件名（UTF-8，最长 40 字节）
    byte fileType;         // 1 B，类型：1=文件, 2=目录
    byte reserved;         // 1 B，保留
    short nameLen;         // 2 B，文件名实际长度
}
```

**目录的实现：** 目录本身也是一个"文件"，占用一个 i-node，其数据块中存放的是 `DirectoryEntry` 数组。

### 2.6 文件描述符 (FileDescriptor)

```java
public class FileDescriptor {
    int fd;                // 文件描述符编号
    int inodeNumber;       // 对应 i-node 号
    int offset;            // 当前读写偏移量
    int mode;              // 打开模式: 0=只读, 1=只写, 2=读写
    boolean dirty;         // 是否有未写回的数据
}
```

系统维护一个 **文件描述符表**，最多同时打开 64 个文件。

### 2.7 路径结构 (PathInfo)

```java
public class PathInfo {
    int parentDirInode;    // 父目录的 i-node 号
    String fileName;       // 文件/目录名
    boolean isAbsolute;    // 是否为绝对路径
    String[] components;   // 路径分解后的各分量
}
```

## 3. 模块详细设计

### 3.1 虚拟磁盘驱动层 — `DiskDriver`

**职责：** 管理底层字节数组，提供盘块级的读写，处理持久化的保存与加载。

```
┌─────────────────────────────┐
│        DiskDriver           │
├─────────────────────────────┤
│ - byte[] disk               │  ← 模拟虚拟磁盘 (16MB)
│ - String imageFilePath      │  ← 持久化文件路径
├─────────────────────────────┤
│ + readBlock(blockNum): byte[]              │
│ + writeBlock(blockNum, data): void         │
│ + readByte(blockNum, offset): byte         │
│ + writeByte(blockNum, offset, value): void │
│ + saveToFile(): boolean                    │
│ + loadFromFile(): boolean                  │
│ + format(): void                           │
│ + getTotalBlocks(): int                    │
│ + getBlockSize(): int                      │
└─────────────────────────────┘
```

**关键方法说明：**

| 方法                    | 逻辑                                                 |
| ----------------------- | ---------------------------------------------------- |
| `readBlock(n)`        | 从 `disk[n * blockSize]` 读取 512 字节             |
| `writeBlock(n, data)` | 将 512 字节写入 `disk[n * blockSize]`              |
| `saveToFile()`        | 将整个 `disk` 字节数组写入 `filesystem.img` 文件 |
| `loadFromFile()`      | 从 `filesystem.img` 读取到 `disk` 数组，校验魔数 |
| `format()`            | 创建空的 `disk` 数组，初始化超级块和位图           |

### 3.2 位图管理层 — `BitmapManager`

**职责：** 维护位图数据，分配和回收盘块。

```
┌───────────────────────────────┐
│         BitmapManager         │
├───────────────────────────────┤
│ - DiskDriver diskDriver       │
├───────────────────────────────┤
│ + allocateBlock(): int        │  ← 分配一个空闲块，返回块号，-1 表示失败
│ + freeBlock(blockNum): void   │  ← 回收一个盘块
│ + allocateBlocks(count): int[]│  ← 批量分配
│ + isBlockFree(blockNum): bool │  ← 检查某块是否空闲
│ + getFreeBlockCount(): int    │  ← 获取空闲块总数
│ + loadBitmap(): void          │  ← 从磁盘加载位图
│ + saveBitmap(): void          │  ← 将位图写回磁盘
│ + initBitmap(): void          │  ← 初始化位图（标记超块/位图/i-node 区为已分配）
└───────────────────────────────┘
```

**位图操作算法：**

```
 allocateBlock():
 1. 扫描位图，找到第一个值为 0 的比特位
 2. 将该比特位设为 1
 3. 减少空闲块计数
 4. 返回对应的块号
 5. 若无空闲块，返回 -1 (磁盘已满)

 freeBlock(blockNum):
 1. 将 blockNum 对应比特位设为 0
 2. 增加空闲块计数
```

### 3.3 索引节点管理层 — `InodeManager`

**职责：** i-node 的分配、释放、读写操作，间接块指针管理。

```
┌─────────────────────────────────┐
│          InodeManager           │
├─────────────────────────────────┤
│ - DiskDriver diskDriver         │
│ - BitmapManager bitmapMgr       │
│ - InodeCache inodeCache         │  ← i-node 缓存 (LRU)
├─────────────────────────────────┤
│ + allocInode(type): int         │  ← 分配一个 i-node，返回编号
│ + freeInode(inodeNum): void     │  ← 释放 i-node
│ + readInode(inodeNum): Inode    │  ← 读取 i-node 数据
│ + writeInode(inode): void       │  ← 写回 i-node 数据
│ + getBlockPointer(inode, logicBlockNum): int     │  ← 获取逻辑块号对应的物理块号
│ + setBlockPointer(inode, logicBlockNum, physBlockNum): void  │
│ + expandFile(inode, targetSize): boolean         │  ← 扩展文件
│ + shrinkFile(inode, targetSize): boolean         │  ← 收缩/截断文件
│ + freeAllBlocks(inode): void    │  ← 释放文件所有盘块
│ + inodeToBytes(inode): byte[]   │  ← 序列化 i-node
│ + bytesToInode(data): Inode     │  ← 反序列化 i-node
└─────────────────────────────────┘
```

**间接块寻址算法：**

```
 getBlockPointer(inode, logicBlockNum):
 1. if logicBlockNum < 10:
        return inode.direct[logicBlockNum]          // 直接块
 2. elif logicBlockNum < 10 + 128:
        // 一级间接块
        indirectBlock = inode.singleIndirect
        offset = logicBlockNum - 10
        return readIndirectBlock(indirectBlock)[offset]
 3. elif logicBlockNum < 10 + 128 + 128*128:
        // 二级间接块
        doubleIndirect = inode.doubleIndirect
        offset = logicBlockNum - 10 - 128
        level1 = offset / 128
        level2 = offset % 128
        indirectBlock = readIndirectBlock(doubleIndirect)[level1]
        return readIndirectBlock(indirectBlock)[level2]
 4. else: 返回 -1 (超出文件大小)
```

### 3.4 文件分配层 — `FileAllocator`

**职责：** 为上层的文件操作提供盘块分配与回收服务，协调 BitmapManager 和 InodeManager。

```
┌─────────────────────────────────┐
│         FileAllocator           │
├─────────────────────────────────┤
│ - BitmapManager bitmapMgr       │
│ - InodeManager inodeMgr         │
├─────────────────────────────────┤
│ + allocBlocksForFile(inodeNum, size): boolean   │  ← 为文件分配所需盘块
│ + freeBlocksOfFile(inodeNum): void              │  ← 释放文件所有盘块
│ + readFileData(inodeNum, offset, length): byte[]│  ← 读取文件数据
│ + writeFileData(inodeNum, offset, data): int    │  ← 写入文件数据
│ + truncateFile(inodeNum, newSize): boolean      │  ← 截断文件
└─────────────────────────────────┘
```

**写文件数据算法：**

```
 writeFileData(inodeNum, offset, data):
 1. inode = inodeMgr.readInode(inodeNum)
 2. 计算写入后的新文件大小: newSize = offset + data.length
 3. if newSize > inode.fileSize:
        inodeMgr.expandFile(inode, newSize)        // 分配新盘块
 4. 按逻辑块号遍历要写入的每一块:
    a. 物理块号 = inodeMgr.getBlockPointer(inode, logicBlockNum)
    b. 将 data 对应部分写入该物理块
 5. 更新 inode.fileSize、modifyTime
 6. inodeMgr.writeInode(inode)
```

### 3.5 路径解析器 — `PathResolver`

**职责：** 路径字符串解析、绝对/相对路径转换、路径合法性检查。

```
┌───────────────────────────────┐
│         PathResolver          │
├───────────────────────────────┤
│ - DirectoryManager dirMgr     │  ← 引用目录管理器
├───────────────────────────────┤
│ + parsePath(path): PathInfo                    │  ← 解析路径字符串
│ + resolveAbsolutePath(path, cwd): PathInfo     │  ← 将路径转为绝对路径信息
│ + validateFileName(name): boolean              │  ← 验证文件名合法性
│ + splitPath(path): String[]                    │  ← 将路径分割为分量
│ + getParentPath(path): String                  │  ← 获取父目录路径
│ + isAbsolutePath(path): boolean                │  ← 判断是否为绝对路径
│ + normalizePath(path): String                  │  ← 规范化路径（去除 ../ 等）
└───────────────────────────────┘
```

**路径解析算法：**

```
 parsePath(path):
 1. 将路径按 '/' 分割为分量数组 components[]
 2. 去除空分量和 '.' 分量
 3. 处理 '..' 分量（上跳一级）
 4. 判断是否为绝对路径（以 '/' 开头）
 5. 返回 PathInfo { components, isAbsolute }
```

### 3.7 文件操作接口层 — `FileSystem`

```
│                                 │

│ + load(): boolean               │  ← 从磁盘文件恢复
│ + exit(): void                  │  ← 保存并退出
│ + info(): SystemInfo            │  ← 系统信息（总块数、空闲块数等）
└─────────────────────────────────┘
```

**文件描述符表管理：**

```
 open(path, mode):
 1. 路径解析 → 找到目标文件的 i-node
 2. 检查文件是否存在且为文件类型
 3. 检查打开模式兼容性
 4. 在 fdTable 中找一个空位
 5. 初始化 FileDescriptor { fd, inodeNumber, offset=0, mode }
 6. 返回 fd 编号

 close(fd):
 1. 检查 fd 是否合法且在打开状态
 2. 若 fd.dirty，写回缓存数据
 3. 清空 fdTable[fd]

 read(fd, size):
 1. 检查 fd 打开模式（必须可读）
 2. 获取 inode，读取 offset 处 size 字节数据
 3. 更新 offset
 4. 返回读取的字节数组

 write(fd, data):
 1. 检查 fd 打开模式（必须可写）
 2. 获取 inode，在 offset 处写入 data
 3. 更新 offset 和 fileSize
 4. 若需实时写回，写入磁盘
 5. 返回写入字节数
```

#### 8.3.1 全流程场景测试

```java
@SpringBootTest
class FullWorkflowTest {

    @Autowired
    private FileSystem fileSystem;

    @Test
    void testFullWorkflow() {
        // 1. 格式化
        fileSystem.format();

        // 2. 创建目录结构
        fileSystem.mkdir("/home");
        fileSystem.mkdir("/home/user1");
        fileSystem.mkdir("/home/user1/docs");
        fileSystem.mkdir("/home/user1/photos");
        fileSystem.mkdir("/home/user2");

        // 3. 切换到 user1
        fileSystem.cd("/home/user1");
        assertEquals("/home/user1", fileSystem.pwd());

        // 4. 创建文件
        fileSystem.create("test.txt");

        // 5. 打开并写入
        int fd = fileSystem.open("test.txt", "rw");
        fileSystem.write(fd, "Hello, 文件系统!".getBytes());
        fileSystem.close(fd);

        // 6. 读取验证
        fd = fileSystem.open("test.txt", "r");
        byte[] data = fileSystem.read(fd, 100);
        assertEquals("Hello, 文件系统!", new String(data).trim());
        fileSystem.close(fd);

        // 7. 复制文件
        fileSystem.copy("/home/user1/test.txt", "/home/user2/test.txt");

        // 8. 验证复制结果
        fd = fileSystem.open("/home/user2/test.txt", "r");
        byte[] copied = fileSystem.read(fd, 100);
        assertEquals("Hello, 文件系统!", new String(copied).trim());
        fileSystem.close(fd);

        // 9. 重命名
        fileSystem.rename("/home/user2/test.txt", "/home/user2/readme.md");

        // 10. 删除源文件
        fileSystem.delete("/home/user1/test.txt");

        // 11. 保存并重新加载
        fileSystem.save();
        fileSystem.load();

        // 12. 验证持久化后的状态
        fileSystem.cd("/home/user2");
        List<EntryInfo> entries = fileSystem.ls(".");
        assertEquals(1, entries.size());
        assertEquals("readme.md", entries.get(0).getName());

        // 13. 树形结构
        String tree = fileSystem.tree("/");
        System.out.println(tree);
        assertTrue(tree.contains("readme.md"));
    }
}
```
