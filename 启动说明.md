# 文件系统管理模拟器 — 启动说明

---

## 1. Java 环境配置

### 1.1 检查是否已安装 JDK

```bash
java -version
javac -version
```

如果看到类似 `java version "23.0.2" 2025-01-21` 的输出，说明已安装。

### 1.2 配置 JAVA_HOME（如未配置）

JDK 通常安装在 `C:\Program Files\Java\` 下，如 `jdk-23`。

**Windows 配置方法：**

1. 打开"系统属性" → "高级" → "环境变量"
2. 在"系统变量"中点击"新建"：
   - 变量名：`JAVA_HOME`
   - 变量值：`C:\Program Files\Java\jdk-23`（以实际版本为准）
3. 找到 `Path` 变量，双击编辑，点击"新建"，添加：
   ```
   %JAVA_HOME%\bin
   ```
4. 保存后打开新终端，运行 `java -version` 验证

**命令行临时配置（当前会话有效）：**

```bash
set JAVA_HOME=C:\Program Files\Java\jdk-23
set PATH=%JAVA_HOME%\bin;%PATH%
```

## 2. 项目启动

### 2.1 启动 REST API 服务（必须先启动）

```bash
cd filesystem-core
java -cp build/classes com.fs.FsApiApp --port 8080
```

参数说明：
- `--port`：服务端口，默认 8080
- `--img`：磁盘映像文件路径，默认 `filesystem.img`

启动后访问 http://localhost:8080/api 验证。

### 2.2 启动 Vue 前端

```bash
cd filesystem-vue
npm run dev
```

启动后在浏览器打开 http://localhost:5173（或终端提示的其他端口）。

> **注意：** 必须先启动 API 服务，再启动前端。前端通过 `http://localhost:8080/api` 与后端通信。

### 2.3 CLI 模式（无需前端）

如不想启动前端，可以直接使用命令行交互界面：

```bash
cd filesystem-core
java -cp build/classes com.fs.FsCliApp
```

CLI 支持全部 22 条命令，输入 `help` 查看所有命令及用法。

## 3. 操作指南

### 3.1 目录树（左侧面板）

| 操作 | 行为 |
|------|------|
| 点击 **▶/▼ 三角** | 展开/折叠子目录（不导航） |
| 点击 **📁 目录名** | 进入该目录 |
| **右键** 目录 | 菜单：进入 / 重命名 / 新建子目录 / 新建文件 / 属性 / 删除 |
| **右键** 文件 | 菜单：打开 / 重命名 / 复制 / 移动 / 属性 / 删除 |

### 3.2 文件列表（右侧上方面板）

| 操作 | 行为 |
|------|------|
| **左键单击** 目录行 | 进入该目录 |
| **左键单击** 文件行 | 在下方预览面板打开文件 |
| **右键** 文件/目录行 | 弹出操作菜单 |
| **右键** 列表空白处 | 全局菜单：新建目录 / 新建文件 / 刷新 / 保存 / 信息 |

### 3.3 文件预览/编辑（右侧中方面板）

- 打开文件后在文本区编辑内容
- 点击 **💾 保存** 按钮将内容写回文件
- 点击 **✖ 关闭** 关闭预览

### 3.4 命令控制台（底部面板）

- 输入命令后按回车执行
- 输入 `help` 查看所有命令及用法
- 支持的命令：

```
系统:   format  save  info  exit
目录:   mkdir  rmdir  ls  cd  pwd  tree
文件:   create  open  close  delete
附加:   rename  copy  move  stat
```

### 3.5 顶部工具栏

| 按钮 | 功能 |
|------|------|
| 💾 保存 | 手动将虚拟磁盘保存到 `filesystem.img` |
| ℹ️ 信息 | 查看系统信息（总块数/空闲块数/使用率等） |
| ⚠️ 格式化 | 重置虚拟磁盘（所有数据将丢失，需确认） |

### 3.6 路径导航栏

- 显示当前路径的面包屑导航
- 点击路径段可直接跳转
- 右侧按钮：新建文件 / 新建目录 / ℹ️ 属性 / 🔄 刷新

## 4. 持久化说明

系统采用 **自动保存 + 手动保存** 结合的方式：

| 触发方式 | 说明 |
|---------|------|
| **退出时** | REST API 服务（FsApiApp）注册了 JVM 关闭钩子，退出时自动保存 |
| **手动保存** | 点击顶部工具栏「💾 保存」按钮，或 CLI 中执行 `save` 命令 |
| **启动恢复** | 启动时自动检测 `filesystem.img`，存在则恢复，不存在则新建 |

验证持久化：
1. 启动服务，创建文件并写入内容
2. 点击「保存」或执行 `save` 命令
3. 停止服务（Ctrl+C）
4. 重新启动服务
5. 查看文件内容是否保留

## 5. 常见问题

| 问题 | 解决 |
|------|------|
| `Address already in use: bind` | 端口被占用，使用 `taskkill //F //IM java.exe` 杀掉旧进程 |
| Vue 前端连不上 API | 确认 API 服务已启动且在 8080 端口；检查 Network 面板是否有 CORS 错误 |
| 格式化提示确认 | `store.format()` 内含 `confirm()` 确认，放心中间不会丢失数据 |
| 如何修改 API 端口 | 启动时加 `--port 端口号`，并修改 `filesystem-vue/src/api/fileSystem.js` 中的 `baseURL` |
